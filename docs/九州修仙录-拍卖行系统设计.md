# 九州修仙录 - 拍卖行系统设计

> 版本：v0.1  
> 日期：2026-02-01  
> 目标：设计完整的拍卖行系统，支持一口价、竞拍、寄售、搜索筛选与交易安全。

## 1. 设计原则

- **经济调控**：通过税率、手续费、价格区间控制经济流通
- **公平交易**：防止恶意刷价、串通竞拍、机器人扫货
- **便捷体验**：强大的搜索筛选、收藏关注、价格提醒
- **安全可控**：交易日志、异常检测、回滚机制
- **社交属性**：查看卖家信息、交易评价、黑名单

## 2. 交易模式

### 2.1 一口价（Buy Now）

| 特性 | 说明 |
|------|------|
| 定价方式 | 卖家设定固定价格 |
| 成交方式 | 买家直接购买，即时成交 |
| 适用场景 | 常规物品、快速出手 |
| 上架时长 | 12/24/48小时可选 |

### 2.2 竞拍（Auction）

| 特性 | 说明 |
|------|------|
| 定价方式 | 起拍价 + 一口价（可选） |
| 成交方式 | 最高出价者获得 |
| 加价规则 | 最低加价幅度（按当前价百分比） |
| 延时机制 | 最后5分钟有出价则延长5分钟 |
| 适用场景 | 稀有物品、价值不确定物品 |

### 2.3 寄售（Consignment）

| 特性 | 说明 |
|------|------|
| 定价方式 | 系统参考价 ± 浮动范围 |
| 成交方式 | 系统撮合，按价格排序 |
| 适用场景 | 大宗材料、标准化物品 |
| 特点 | 快速成交、价格稳定 |

| end_time | timestamptz | 结束时间 |

| level | int | 物品等级（冗余） |

## 4. 费用体系

### 4.1 上架费

| 时长 | 费用（按底价百分比） |
|------|----------------------|
| 12小时 | 1% |
| 24小时 | 2% |
| 48小时 | 3% |

### 4.2 成交税

| 物品类型 | 税率 |
|----------|------|
| 普通物品 | 5% |
| 装备 | 8% |
| 稀有物品 | 10% |

### 4.3 特殊规则

- 流拍退还50%上架费
- 主动下架不退上架费
- 宗门成员交易税率减免（根据宗门等级）

## 5. 搜索与筛选

### 5.1 筛选维度

| 维度 | 选项 |
|------|------|
| 分类 | 装备/消耗品/材料/功法/其他 |
| 品质 | 黄/玄/地/天 |
| 等级范围 | 1-100 |
| 价格范围 | 自定义 |
| 交易类型 | 一口价/竞拍/寄售 |
| 装备槽位 | 武器/头部/衣服等 |
| 词条筛选 | 包含特定词条 |

### 5.2 排序方式

- 价格（升/降）
- 剩余时间（升/降）
- 品质（升/降）
- 上架时间（最新）
- 出价次数（热门）

## 6. 安全机制

### 6.1 价格保护

- 最低价：不低于系统回收价的50%
- 最高价：不高于参考价的500%
- 异常检测：短时间大量低价交易触发审核

### 6.2 交易限制

- 每日上架数量限制（根据VIP等级）
- 同一物品重复上架冷却
- 自买自卖检测与禁止
- IP/设备关联账号交易监控

### 6.3 争议处理

- 交易完成后24小时内可申诉
- 系统自动记录完整交易日志
- 异常交易可回滚

## 7. API设计

### 7.1 上架物品

```
POST /api/auction/list
Body: { 
  item_instance_id, 
  listing_type, 
  buyout_price?, 
  start_price?, 
  duration_hours 
}
```

### 7.2 搜索物品

```
GET /api/auction/search
Query: { 
  keyword?, category?, quality?, 
  price_min?, price_max?, 
  listing_type?, sort?, page, limit 
}
```

### 7.3 购买/出价

```
POST /api/auction/buy
Body: { listing_id }

POST /api/auction/bid
Body: { listing_id, bid_price }
```

### 7.4 我的交易

```
GET /api/auction/my/listings
GET /api/auction/my/bids
GET /api/auction/my/history
```
