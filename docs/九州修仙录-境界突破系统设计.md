# 九州修仙录 · 境界突破系统设计说明

## 一、设计目标

- 为角色提供一条清晰的长期纵向成长主线，从「凡人」一路晋升高阶修真境界。
- 强化境界与功法、秘境、副本、任务、掉落等系统的联动，让突破不只是数值成长，而是阶段性内容解锁点。
- 兼容当前项目已有数据结构（characters 表、功法系统、秘境系统等），以尽量少的结构改动实现可玩性较高的玩法。

---

## 二、现有相关系统概览

### 2.1 角色基础数据

- 表：`characters`
- 关键字段：
  - `realm`：主境界，当前默认 `'凡人'`
  - `sub_realm`：子境界（当前使用较少，预留细分空间）
  - `exp`：角色经验
  - `spirit_stones`：灵石（核心货币之一）
  - 其他战斗属性：`qixue/max_qixue/lingqi/max_lingqi/wugong/fagong/wufang/fafang/...`

### 2.2 境界顺序（已有）

在组队、秘境等服务中已定义统一境界顺序，用于比较高低：

```text
凡人
炼精化炁·养气期
炼精化炁·通脉期
炼精化炁·凝炁期
炼炁化神·炼己期
炼炁化神·采药期
炼炁化神·结胎期
炼神返虚·养神期
炼神返虚·还虚期
炼神返虚·合道期
炼虚合道·证道期
炼虚合道·历劫期
炼虚合道·成圣期
```

所有境界突破逻辑应严格基于该顺序进行比较与推进。

### 2.3 功法与境界的初步关系（已有）

在功法系统中，已存在一次“自动境界提升”的实现：

- 当角色修炼功法 `tech-yangqi-jue` 至 **第 2 层** 且当前境界为 `凡人` 时：
  - 自动将 `characters.realm` 更新为 `炼精化炁·养气期`
  - 并向任务系统上报 `upgrade_realm` 类型进度

说明：

- 目前“凡人 → 炼精化炁·养气期”的提升由功法驱动，属于初版示例。
- `technique_layer` 配置中大量存在 `required_realm` 字段，高阶功法层数需要更高境界。

### 2.4 相关材料与掉落（已有）

在物品配置中已经存在许多“突破感很强”的材料，例如：

- `mat-wudao-yujian`：悟道玉简 —— 功法突破瓶颈材料
- `mat-daoyun-jinghe`：道韵晶核 —— 功法终极突破材料

境界突破系统将参考这些设计，额外引入少量“境界专用丹药/素材”，并与秘境掉落、任务奖励等挂钩。

## 三、境界体系与阶段划分

基于现有顺序，将境界划分为若干阶段，方便数值和玩法规划：

1. **凡人阶段**
   - 凡人

2. **炼精化炁阶段**
   - 炼精化炁·养气期
   - 炼精化炁·通脉期
   - 炼精化炁·凝炁期

3. **炼炁化神阶段**
   - 炼炁化神·炼己期
   - 炼炁化神·采药期
   - 炼炁化神·结胎期

4. **炼神返虚阶段**
   - 炼神返虚·养神期
   - 炼神返虚·还虚期
   - 炼神返虚·合道期

5. **炼虚合道阶段**
   - 炼虚合道·证道期
   - 炼虚合道·历劫期
   - 炼虚合道·成圣期

当前版本建议重点落地：

- 从 **凡人 → 炼精化炁·凝炁期** 的完整链路；
- 并预留从 **炼精化炁·凝炁期 → 炼炁化神·炼己期** 的突破入口，用于后续扩展高级玩法。

## 四、突破入口与交互流程

### 4.1 客户端入口规划

建议提供两个常用入口：

1. **角色面板入口**
   - 在角色信息区域显示当前境界（文字与小图标）。
   - 境界下方增加按钮【境界突破】。
   - 当条件不足时，按钮置灰并显示“查看条件”；条件满足时高亮。

2. **功法面板联动入口**
   - 当某功法下一层的 `required_realm` 高于当前境界时，在功法详情中提示：
     - “需要境界达到 XXX 才能继续修炼”；
     - 并提供跳转到“境界突破”界面的入口。

### 4.2 服务端接口建议

可新增统一接口：

- `POST /api/realm/breakthrough`
- 入参示例：
  - 简版：`{ targetRealm: string }` 或 `{ direction: "next" }`
- 返回：
  - `success: boolean`
  - `message: string`
  - `data`（成功时）：新境界、属性变更、获得的属性点、可能的奖励提示等

具体技术实现可放在单独的 `realmService` 中，调用 `characters`、`character_technique`、秘境记录等数据。

### 4.3 基础流程（无失败版）

以“版本 1：尝试即成功，只要满足条件”为前提：

1. 玩家打开突破界面，服务器返回当前境界和“下一境界”的条件列表与消耗预览。
2. 客户端根据返回信息展示进度（满足/未满足）。
3. 玩家点击【突破】，向服务端发起突破请求。
4. 服务端依次校验：
   - 是否存在下一境界；
   - 是否满足经验、功法层数、秘境通关、材料与灵石等条件；
   - 是否处于可突破状态（例如没有在战斗中、没有处于某些禁制状态）。
5. 若校验通过：
   - 扣除经验 / 灵石 / 材料；
   - 更新 `characters.realm`（和必要时的 `sub_realm`）；
   - 应用境界带来的属性/上限变更；
   - 记录日志，并向任务/成就等系统上报。
6. 返回成功结果，客户端播放突破动画、飘字、语音等，强化仪式感。

> 后续版本可以在高阶境界（例如炼炁化神之后）引入失败概率与惩罚机制，前期保持 100% 成功保证成长节奏顺畅。

## 五、突破条件与资源消耗设计

### 5.1 共用条件类型

每次境界突破可以由以下几类条件组合构成：

1. **经验条件**
   - `characters.exp` 达到一定阈值；
   - 可选择是否扣除一部分经验作为突破消耗。

2. **功法修炼条件**
   - 至少一门“主修功法”达到指定层数；
   - 或若干功法之和达到一定总层数；
   - 技术上基于 `character_technique` 与 `technique_layer` 实现。

3. **秘境/副本条件**
   - 通关对应境界段位的试炼秘境；
   - 使用秘境系统中的境界限制与通关记录进行校验。

4. **材料与货币消耗**
   - 灵石：`spirit_stones` 一定数量；
   - 丹药：新建或复用对应境界的专属丹药，如“通脉丹”“凝炁丹”“炼己丹”等；
   - 稀有材料：部分高阶境界要求使用稀有材料（如悟道类、道韵类素材）。

5. **冷却与次数限制（可选，后期扩展）**
   - 每日可尝试突破次数上限；
   - 或失败后有冷却 CD（在高阶境界引入）。

### 5.2 典型数值框架（示意）

以下为示例，具体数值由策划后期根据整体节奏微调：

1. **凡人 → 炼精化炁·养气期**
   - 条件：
     - 经验 `exp ≥ 1000`
     - 功法 `tech-yangqi-jue` ≥ 2 层
   - 消耗：
     - 扣除经验 500（或仅作阈值，不扣除）
     - 无材料与灵石消耗（新手友好）
   - 说明：
     - 与现有“功法升到 2 层自动提升境界”的逻辑兼容，可将经验阈值检查放在功法升级处。

2. **炼精化炁·养气期 → 炼精化炁·通脉期**
   - 条件：
     - 经验 `exp ≥ 5000`
     - 至少 1 门基础战斗功法（刀法/拳法/符术等）≥ 3 层
     - 通关一次对应的“气血修炼秘境”低难度关卡
   - 消耗：
     - 灵石 100
     - 1 颗“通脉丹”（秘境与周常产出）

3. **炼精化炁·通脉期 → 炼精化炁·凝炁期**
   - 条件：
     - 经验 `exp ≥ 15000`
     - 至少 2 门功法 ≥ 4 层
     - 通关秘境更高难度关卡一次
   - 消耗：
     - 灵石 300
     - 1 颗“凝炁丹”
     - 少量功法残页类材料（如 `mat-gongfa-canye`）

4. **炼精化炁·凝炁期 → 炼炁化神·炼己期**
   - 条件：
     - 经验 `exp ≥ 40000`
     - 主修功法达到 5 层，且满足其 `required_realm` 前置要求；
     - 完成一条主线任务“境界考验”。
   - 消耗：
     - 灵石 1000
     - 1 颗“炼己丹”
     - 指定秘境掉落的特殊材料若干。

> 后续更高境界可以沿用相同结构，主要通过提高经验与消耗、叠加秘境与宗门门槛来控制节奏。

## 六、突破成功效果与数值收益

### 6.1 收益类型

每一次境界突破可带来下列一部分或全部收益：

1. **基础属性提升**
   - 最大气血：`max_qixue` 增加（百分比或固定值）
   - 最大灵气：`max_lingqi` 增加
   - 攻防属性：`wugong/fagong/wufang/fafang` 等按百分比提升。

2. **可分配属性点**
   - 获得 `attribute_points`，由玩家自由分配 `jing/qi/shen`。

3. **抗性与成长属性**
   - 增加 `kongzhi_kangxing`、五行抗性等高阶属性。

4. **系统解锁**
   - 解锁更高阶秘境难度；
   - 解锁宗门功能、炼丹炼器等级上限等。

5. **功法与技能上限**
   - 满足功法 layer 的 `required_realm` 要求，可以继续修炼高阶层数；
   - 部分技能的进阶版本可在特定境界后开放。

### 6.2 示意数值（可调）

以主要境界为例：

- **凡人 → 炼精化炁·养气期**
  - `max_qixue` +10%
  - `max_lingqi` +10%
  - `attribute_points` +5

- **炼精化炁·养气期 → 通脉期**
  - `max_qixue` +15%
  - `max_lingqi` +15%
  - `wugong/fagong` +5%
  - `attribute_points` +5

- **炼精化炁·通脉期 → 凝炁期**
  - `max_qixue` +15%
  - `max_lingqi` +15%
  - 攻击/防御各 +5%～8%
  - `attribute_points` +10

- **炼精化炁·凝炁期 → 炼炁化神·炼己期**
  - `max_qixue` +20%
  - `max_lingqi` +20%
  - 攻击/防御属性 +10%
  - `kongzhi_kangxing` +3%
  - `attribute_points` +15

> 实际数值需结合当前怪物强度、功法加成、秘境收益等整体平衡后确定。

## 七、与其他系统的联动

### 7.1 与功法系统

- 高阶功法层数通过 `required_realm` 锁定在某个境界之后；
- 境界突破前：
  - 功法界面高亮“已达本境界功法上限，请先进行境界突破”；
- 境界突破后：
  - 提醒玩家相关功法可以继续修炼；
  - 触发主线或成就记录“完成某境界突破”的节点。

### 7.2 与秘境系统

- 秘境入口设置最低境界要求：
  - 例如某些秘境只允许“炼精化炁·通脉期及以上”进入；
- 境界突破条件中绑定特定秘境：
  - 某些境界需要通关对应试炼秘境，保证玩家实际战斗能力跟得上境界。

### 7.3 与任务/成就系统

- 主线任务：
  - 在关键境界（如炼精化炁起点、炼炁化神起点等）设计专门主线节点；
  - 突破完成后更新任务进度。
- 成就系统：
  - 记录“不降境界完成 XX 战斗”、“首位达到某高境界”等成就。

### 7.4 与掉落和经济系统

- 境界突破所需丹药与材料主要来自：
  - 对应境界的秘境 Boss 掉落；
  - 周常活动与宗门商店兑换；
  - 稀有宝箱奖励。
- 灵石消耗控制整体经济节奏，避免资源堆积。

## 八、后续扩展与迭代方向

1. **高境界失败与心魔机制**
   - 在炼炁化神之后开启“突破失败概率”，失败后附加心魔 Debuff；
   - 玩家可通过战斗或秘境净化心魔，恢复状态。

2. **悟道值/机缘系统**
   - 每次突破失败积累“悟道值”，提升下次突破成功率；
   - 特定事件或道具可临时提升成功率或减少消耗。

3. **宗门协助突破**
   - 在宗门大阵中进行突破，可获得成功率加成或“失败保护”；
   - 消耗宗门贡献或宗门专属材料。

4. **世界公告与排行榜**
   - 服务器首个达到某高境界的玩家触发世界公告；
   - 境界排行榜用于展示进度，强化长期竞争目标。
