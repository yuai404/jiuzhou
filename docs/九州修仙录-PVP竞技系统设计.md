# 九州修仙录 - PVP竞技系统设计

> 版本：v0.1  
> 日期：2026-02-01  
> 目标：设计完整的PVP系统，包含竞技场、境界战、排位赛与跨服对战。

## 1. 设计原则

- **公平竞技**：同境界对战，属性压制可控
- **匹配合理**：积分匹配，避免碾压局
- **奖励驱动**：排名奖励、赛季奖励
- **策略深度**：功法搭配、技能选择有意义
- **社交属性**：观战、回放、排行榜

## 2. PVP模式

### 2.1 竞技场（1v1实时对战）

- 按积分段匹配
- 每日免费5次，可购买
- 奖励：积分、荣誉点、赛季奖励

### 2.2 境界战（同境界1v1）

- 属性压制到境界上限
- 纯技术对决
- 奖励：境界荣誉、称号

### 2.3 排位赛（段位系统）

- 段位：青铜→白银→黄金→铂金→钻石→宗师→传说
- 每月赛季重置
- 奖励：段位奖励、赛季皮肤

### 2.4 组队竞技（3v3）

- 车轮战/团战可选
- 按队伍平均积分匹配

## 3. 数据结构

### 3.1 PVP积分表 (pvp_rating)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigserial | 主键 |
| character_id | int | 角色ID |
| mode | varchar(16) | 模式 |
| rating | int | 当前积分 |
| peak_rating | int | 历史最高 |
| rank_tier | varchar(16) | 段位 |
| wins | int | 胜场 |
| losses | int | 负场 |
| season_id | varchar(32) | 赛季ID |

### 3.2 对战记录表 (pvp_match)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | varchar(64) | 对战ID |
| mode | varchar(16) | 模式 |
| player1_id | int | 玩家1 |
| player2_id | int | 玩家2 |
| winner_id | int | 胜者 |
| rating_change | int | 积分变化 |
| battle_log | jsonb | 战斗日志 |
| created_at | timestamptz | 时间 |

## 4. 积分与段位

| 段位 | 积分范围 |
|------|----------|
| 青铜 | 0-999 |
| 白银 | 1000-1499 |
| 黄金 | 1500-1999 |
| 铂金 | 2000-2499 |
| 钻石 | 2500-2999 |
| 宗师 | 3000-3499 |
| 传说 | 3500+ |

## 5. 奖励系统

- 每场胜利：荣誉点×20
- 每场失败：荣誉点×5
- 赛季结算：按段位发放灵石、称号、外观
