# 九州修仙录 - Web挂机多人回合游戏

## 项目概述

一款中国风修仙主题的Web挂机多人回合制游戏，玩家在九州大陆修炼成仙，体验完整的修仙之路。

## 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | React + TypeScript + Vite |
| 状态管理 | Zustand / Redux Toolkit |
| 游戏引擎 | boardgame.io |
| 实时通信 | Socket.io |
| 后端 | Node.js + Express |
| 数据库 | PostgreSQL |
| 缓存 | Redis |
| ORM | Prisma |

游戏无等级概念，以境界划分修为高低：

```
凡人

        ├── 养气期
        ├── 通脉期
        └── 凝炁期
  └── 炼炁化神
        ├── 炼己期
        ├── 采药期
        └── 结胎期
  └── 炼神返虚
        ├── 养神期
        ├── 还虚期
        └── 合道期
  └── 炼虚合道
        ├── 证道期
        ├── 历劫期
        └── 成圣期

每个境界影响：
- 基础属性加成
- 可学习功法等级
- 可进入秘境等级
- 可挑战怪物范围

| 银两 | 基础货币，日常消耗、NPC商店 |

```

- 9个区域，每区域独立场景

## 数据库设计

### 核心表结构

```sql
-- 玩家表
CREATE TABLE players (
  id UUID PRIMARY KEY,
  username VARCHAR(50) UNIQUE,
  password_hash VARCHAR(255),
  realm VARCHAR(50) DEFAULT '凡人',
  sub_realm VARCHAR(50) DEFAULT NULL,
  spirit_stones BIGINT DEFAULT 0,
  silver BIGINT DEFAULT 1000,
  sect_id UUID REFERENCES sects(id),
  created_at TIMESTAMP DEFAULT NOW(),
  last_online TIMESTAMP
);

-- 玩家属性表
CREATE TABLE player_stats (
  player_id UUID PRIMARY KEY REFERENCES players(id),
  hp INT DEFAULT 100,
  max_hp INT DEFAULT 100,
  attack INT DEFAULT 10,
  defense INT DEFAULT 5,
  spirit INT DEFAULT 10,
  speed INT DEFAULT 10
);

-- 功法表
CREATE TABLE techniques (
  id UUID PRIMARY KEY,
  name VARCHAR(100),
  type VARCHAR(20),
  required_realm VARCHAR(50),
  base_power INT,
  description TEXT
);

-- 玩家功法关联
CREATE TABLE player_techniques (
  player_id UUID REFERENCES players(id),
  technique_id UUID REFERENCES techniques(id),
  level INT DEFAULT 1,
  experience INT DEFAULT 0,
  PRIMARY KEY (player_id, technique_id)
);

-- 物品表
CREATE TABLE items (
  id UUID PRIMARY KEY,
  name VARCHAR(100),
  type VARCHAR(50),
  rarity VARCHAR(20),
  effects JSONB,
  description TEXT
);

-- 玩家背包
CREATE TABLE player_inventory (
  player_id UUID REFERENCES players(id),
  item_id UUID REFERENCES items(id),
  quantity INT DEFAULT 1,
  PRIMARY KEY (player_id, item_id)
);

-- 宗门表
CREATE TABLE sects (
  id UUID PRIMARY KEY,
  name VARCHAR(100) UNIQUE,
  level INT DEFAULT 1,
  leader_id UUID,
  treasury_stones BIGINT DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 聊天消息表
CREATE TABLE messages (
  id UUID PRIMARY KEY,
  sender_id UUID REFERENCES players(id),
  channel VARCHAR(20),
  target_id UUID,
  content TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  is_read BOOLEAN DEFAULT FALSE
);

-- 市场交易表
CREATE TABLE market_listings (
  id UUID PRIMARY KEY,
  seller_id UUID REFERENCES players(id),
  item_id UUID REFERENCES items(id),
  quantity INT,
  price_type VARCHAR(20),
  price BIGINT,
  created_at TIMESTAMP DEFAULT NOW(),
  expires_at TIMESTAMP
);
```

### 离线收益计算

```typescript
// server/src/services/idleService.ts
export function calculateOfflineRewards(
  player: Player,
  lastOnline: Date
): OfflineRewards {
  const offlineSeconds = (Date.now() - lastOnline.getTime()) / 1000;
  const maxOfflineHours = 12; // 最大离线收益时间
  const effectiveSeconds = Math.min(offlineSeconds, maxOfflineHours * 3600);
  
  // 根据境界和挂机区域计算收益
  const expPerSecond = getExpRate(player.realm, player.idleArea);
  const lootChance = getLootRate(player.realm, player.idleArea);
  
  return {
    experience: Math.floor(effectiveSeconds * expPerSecond),
    silver: Math.floor(effectiveSeconds * silverRate),
    items: generateLoot(effectiveSeconds, lootChance),
  };
}
```
