# 九州修仙录 - 战斗系统设计

> 版本：v0.1  
> 日期：2026-01-31  
> 目标：设计完整的回合制战斗系统，包含属性公式、伤害计算、控制机制、技能效果等，支持 PVE/PVP 场景。

## 1. 设计原则

- **可预测**：玩家能通过属性面板预估战斗结果，避免"黑箱"体验
- **有深度**：通过属性克制、技能组合、回合节奏提供策略空间
- **可平衡**：所有公式参数可配置，便于后续调参
- **可扩展**：支持新增属性、效果类型、战斗模式
- **高性能**：公式计算简洁，适合服务端批量结算

## 4. 伤害计算公式

### 4.1 基础伤害公式

```
基础伤害 = 技能系数 × 攻击力 + 技能固定值
```

- 物理技能：`攻击力 = wugong`
- 法术技能：`攻击力 = fagong`
- 混合技能：`攻击力 = max(wugong, fagong)` 或按配置比例

### 4.2 防御减伤公式

采用**递减公式**，避免防御堆叠导致无敌：

```
防御减伤率 = 防御力 / (防御力 + 防御常数)
实际伤害 = 基础伤害 × (1 - 防御减伤率)
```

**防御常数建议值**（按境界分段）：

| 境界 | 防御常数 | 说明 |
|------|----------|------|
| 凡人 | 50 | 10防御 ≈ 16.7%减伤 |
| 炼精化炁 | 100 | 20防御 ≈ 16.7%减伤 |
| 炼炁化神 | 200 | 40防御 ≈ 16.7%减伤 |
| 炼神返虚 | 400 | 80防御 ≈ 16.7%减伤 |
| 炼虚合道 | 800 | 160防御 ≈ 16.7%减伤 |

### 4.3 命中与闪避

```
实际命中率 = clamp(攻击方命中 - 防守方闪避, 2000, 10000)
```

- 最低命中率：20%（保底）
- 最高命中率：100%

**判定流程**：
1. 随机数 [0, 10000)
2. 若 < 实际命中率，则命中
3. 否则 MISS，伤害为 0

### 4.4 招架判定

命中后触发招架判定：

```
招架触发率 = 防守方招架
招架减伤 = 30%（固定）
```

**判定流程**：
1. 随机数 [0, 10000)
2. 若 < 招架触发率，则招架成功
3. 招架成功：伤害 × 0.7

### 4.5 暴击判定

```
实际暴击率 = clamp(攻击方暴击 - 防守方抗暴, 0, 10000)
暴击伤害 = 伤害 × (攻击方爆伤 / 10000)
```

**判定流程**：
1. 随机数 [0, 10000)
2. 若 < 实际暴击率，则暴击
3. 暴击时伤害乘以爆伤倍率

### 4.6 五行克制

```
克制加成 = 15%（1500万分比）
被克减伤 = 对应五行抗性
```

**克制关系**：
- 金 → 木 → 土 → 水 → 火 → 金

**计算**：
```
若攻击方元素克制防守方元素：
  伤害 = 伤害 × 1.15
  
五行减伤 = 伤害 × (对应抗性 / 10000)
最终伤害 = 伤害 - 五行减伤
```

### 4.7 最终伤害公式

```
最终伤害 = 基础伤害
         × (1 - 防御减伤率)
         × 命中判定(0或1)
         × 招架判定(0.7或1)
         × 暴击判定(爆伤/10000 或 1)
         × (1 + 攻击方增伤/10000)
         × 五行克制(1.15或1)
         × (1 - 五行抗性/10000)
```

### 4.8 伤害计算代码示例

```typescript
interface DamageResult {
  damage: number;           // 最终伤害
  isMiss: boolean;          // 是否闪避
  isParry: boolean;         // 是否招架
  isCrit: boolean;          // 是否暴击
  isElementBonus: boolean;  // 是否五行克制
}

function calculateDamage(
  attacker: BattleUnit,
  defender: BattleUnit,
  skill: Skill
): DamageResult {
  const result: DamageResult = {
    damage: 0,
    isMiss: false,
    isParry: false,
    isCrit: false,
    isElementBonus: false,
  };

  // 1. 基础伤害
  const atkPower = skill.damageType === 'physical' 
    ? attacker.wugong 
    : attacker.fagong;
  let damage = skill.coefficient * atkPower + skill.fixedDamage;

  // 2. 命中判定
  const hitRate = Math.max(2000, Math.min(10000, 
    attacker.mingzhong - defender.shanbi));
  if (Math.random() * 10000 >= hitRate) {
    result.isMiss = true;
    return result;
  }

  // 3. 防御减伤
  const defense = skill.damageType === 'physical' 
    ? defender.wufang 
    : defender.fafang;
  const defenseConstant = getDefenseConstant(defender.realm);
  const defenseReduction = defense / (defense + defenseConstant);
  damage *= (1 - defenseReduction);

  // 4. 招架判定
  if (Math.random() * 10000 < defender.zhaojia) {
    result.isParry = true;
    damage *= 0.7;
  }

  // 5. 暴击判定
  const critRate = Math.max(0, attacker.baoji - defender.kangbao);
  if (Math.random() * 10000 < critRate) {
    result.isCrit = true;
    damage *= attacker.baoshang / 10000;
  }

  // 6. 增伤加成
  damage *= (1 + attacker.zengshang / 10000);

  // 7. 五行克制
  if (isElementCounter(attacker.element, defender.element)) {
    result.isElementBonus = true;
    damage *= 1.15;
  }

  // 8. 五行抗性
  const resistance = getElementResistance(defender, skill.element);
  damage *= (1 - resistance / 10000);

  result.damage = Math.floor(Math.max(1, damage));
  return result;
}
```


## 5. 治疗计算公式

### 5.1 基础治疗

```
基础治疗 = 技能系数 × 法攻 + 技能固定值
```

### 5.2 治疗加成与减疗

```
实际治疗 = 基础治疗 
         × (1 + 施法者治疗加成/10000)
         × (1 - 目标减疗/10000)
```

### 5.3 治疗上限

```
单次治疗上限 = 目标最大气血 × 50%（可配置）
```

## 6. 护盾机制

### 6.1 护盾属性

```typescript
interface Shield {
  id: string;              // 护盾ID
  value: number;           // 护盾值
  maxValue: number;        // 护盾上限
  duration: number;        // 持续回合（-1为永久）
  absorbType: 'all' | 'physical' | 'magic';  // 吸收类型
  priority: number;        // 优先级（高优先消耗）
}
```

### 6.2 护盾结算

```
伤害吸收顺序：按优先级从高到低
剩余伤害 = 原始伤害

for each shield in shields (按priority降序):
  if shield.absorbType 匹配伤害类型:
    absorbed = min(shield.value, 剩余伤害)
    shield.value -= absorbed
    剩余伤害 -= absorbed
    if shield.value <= 0:
      移除护盾

实际扣血 = 剩余伤害
```

## 7. 控制效果

### 7.1 控制类型

| 类型 | 效果 | 可净化 | 递减 |
|------|------|--------|------|
| 眩晕(stun) | 跳过行动 | 是 | 是 |
| 冻结(freeze) | 跳过行动+易伤20% | 是 | 是 |
| 沉默(silence) | 禁止法术技能 | 是 | 否 |
| 缴械(disarm) | 禁止物理技能 | 是 | 否 |
| 定身(root) | 禁止移动（预留） | 是 | 否 |
| 嘲讽(taunt) | 强制攻击施法者 | 是 | 否 |
| 恐惧(fear) | 随机行动 | 是 | 是 |

### 7.2 控制抗性判定

```
控制成功率 = 技能控制率 × (1 - 目标控制抗性/10000)
```

**判定流程**：
1. 随机数 [0, 10000)
2. 若 < 控制成功率，则控制生效
3. 否则抵抗成功

### 7.3 控制递减（PVP）

同类硬控在短时间内递减：

```
第1次：100% 持续时间
第2次：50% 持续时间
第3次：25% 持续时间
第4次及以后：免疫 3 回合
```

递减计数在 5 回合后重置。

## 8. Buff/Debuff 系统

### 8.1 效果结构

```typescript
interface BuffEffect {
  id: string;                    // 效果ID
  name: string;                  // 效果名称
  type: 'buff' | 'debuff';       // 类型
  category: string;              // 分类（见下表）
  
  // 数值效果
  attrModifiers?: Array<{
    attr: string;                // 属性ID
    value: number;               // 数值
    mode: 'flat' | 'percent';    // 加算/乘算
  }>;
  
  // 持续效果
  dot?: {                        // 持续伤害
    damage: number;
    damageType: 'physical' | 'magic' | 'true';
    element?: string;
  };
  hot?: {                        // 持续治疗
    heal: number;
  };
  
  // 控制
  control?: string;              // 控制类型
  
  // 持续
  duration: number;              // 持续回合
  stackable: boolean;            // 是否可叠加
  maxStacks: number;             // 最大层数
  
  // 标签
  tags: string[];                // 用于驱散/免疫判定
  dispellable: boolean;          // 是否可驱散
}
```

### 8.2 效果分类

| 分类 | 说明 | 示例 |
|------|------|------|
| attr_up | 属性提升 | 攻击+10% |
| attr_down | 属性降低 | 防御-20% |
| dot | 持续伤害 | 中毒、灼烧 |
| hot | 持续治疗 | 回春 |
| shield | 护盾 | 金钟罩 |
| control | 控制 | 眩晕、沉默 |
| immunity | 免疫 | 免控、免伤 |
| special | 特殊机制 | 标记、印记 |

### 8.3 叠加规则

```
同ID效果：
  - stackable=true：叠加层数，刷新持续时间
  - stackable=false：刷新持续时间，不叠加数值

不同ID同类效果：
  - 默认共存
  - 可配置互斥组（同组取最高）
```
