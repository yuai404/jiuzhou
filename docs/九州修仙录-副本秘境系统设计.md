# 九州修仙录 - 副本秘境系统设计

> 版本：v0.1  
> 日期：2026-01-31  
> 目标：设计完整的副本秘境系统，支持单人/组队挑战、多层关卡、Boss机制、限时挑战与稀有掉落。

## 1. 设计原则

- **目标导向**：每个秘境有明确的产出定位（功法材料/装备/货币/特殊道具）
- **难度分层**：支持多难度模式，满足不同阶段玩家需求
- **组队激励**：组队挑战有额外奖励，鼓励社交
- **可控节奏**：通过次数限制、体力消耗控制产出速度
- **机制深度**：Boss战有可学习的机制，而非纯数值碾压

## 2. 秘境分类

### 2.1 按玩法类型

| 类型 | 说明 | 特点 |
|------|------|------|
| 材料秘境 | 产出功法升级材料 | 每日限次，稳定产出 |
| 装备秘境 | 产出装备与词条材料 | 周常限次，高价值 |
| 试炼秘境 | 产出经验与境界材料 | 消耗体力，可刷 |
| 挑战秘境 | 产出稀有道具与称号 | 高难度，成就向 |
| 活动秘境 | 限时开放，特殊奖励 | 运营活动 |

### 2.2 按人数规模

| 规模 | 人数 | 说明 |
|------|------|------|
| 单人 | 1 | 个人挑战，难度适中 |
| 小队 | 2-4 | 组队配合，需要分工 |
| 团队 | 5-10 | 大型副本，复杂机制 |

## 3. 秘境结构设计

### 3.1 秘境层级

```
秘境定义 (dungeon_def)
  └── 难度模式 (dungeon_difficulty)
        └── 关卡/层 (dungeon_stage)
              └── 波次 (dungeon_wave)
                    └── 怪物配置 (wave_monster)
```

### 3.2 关卡类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 战斗关 | 击败所有怪物 | 清怪、精英战 |
| Boss关 | 击败Boss | 首领战 |
| 事件关 | 触发特殊事件 | 宝箱、机关、选择 |
| 休息关 | 恢复与准备 | 回血、换装、商店 |

## 6. 奖励系统

### 6.1 奖励类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 通关奖励 | 完成秘境获得 | 基础材料、货币 |
| 首通奖励 | 首次通关获得 | 稀有道具、称号 |
| 评分奖励 | 根据表现评分 | S/A/B/C级奖励 |
| 隐藏奖励 | 满足特殊条件 | 成就、隐藏Boss |
| 组队加成 | 组队额外奖励 | 经验/掉落加成 |

### 6.2 评分规则

```typescript
interface DungeonScore {

  // 伤害评分（DPS达标）
  damageScore: number;    // 0-20分
  
  // 机制评分（正确处理机制）

  // 总分对应等级
  // S: 90-100, A: 75-89, B: 60-74, C: 40-59, D: 0-39
}
```

## 7. 秘境流程

### 7.1 进入流程

```
1. 检查解锁条件（境界、前置任务等）
2. 检查次数限制（日/周）
3. 检查体力消耗
4. 创建秘境实例
5. 组队模式：等待队员准备
6. 进入秘境
```

### 7.2 战斗流程

```
1. 加载关卡配置
2. 生成怪物波次
3. 回合制战斗
4. 波次清空 → 下一波次
5. 关卡清空 → 下一关卡
6. 全部通关 → 结算奖励
```

### 7.3 结算流程

```
1. 计算通关评分
2. 生成掉落物品
3. 分配奖励（组队随机/需求）
4. 更新通关记录
5. 更新次数计数
6. 检查成就/任务
7. 返回结果
```

## 9. API设计

### 9.1 秘境列表

```
GET /api/dungeon/list
Query: category, type, available_only
Response: { dungeons: DungeonInfo[] }
```

### 9.2 秘境详情

```
GET /api/dungeon/:dungeonId
Response: { dungeon: DungeonDetail, entry_count: EntryCount }
```

### 9.3 创建实例

```
POST /api/dungeon/create
Body: { dungeon_id, difficulty_id, team_id? }
Response: { instance_id, instance: DungeonInstance }
```

### 9.4 进入秘境

```
POST /api/dungeon/enter
Body: { instance_id }
Response: { stage: StageInfo, wave: WaveInfo }
```

### 9.5 战斗行动

```
POST /api/dungeon/action
Body: { instance_id, skill_id, target_ids }
Response: { battle_result, stage_status }
```

### 9.6 领取奖励

```
POST /api/dungeon/claim
Body: { instance_id }
Response: { rewards: RewardList, score: ScoreInfo }
```
