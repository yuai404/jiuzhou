# 属性注册表与反弹减伤设计

## 目标

把“暴击伤害减免”和“反弹伤害减免”完整接入角色属性、功法/词条来源、面板展示与战斗结算，并把当前分散在服务端与客户端的属性名单收口为可复用的注册表，避免后续继续在多处手抄字符串。

## 现状

- `jianbaoshang` 已经部分存在于服务端约束、角色计算和战斗暴击公式中，但仍有若干前端/共享名单没有统一复用。
- `reflect_damage` 和 `damage_type: reflect` 已有反弹伤害结算链路，但缺少专门的“反弹伤害减免”属性。
- 属性 key、中文名、百分比属性集合、功法被动白名单、标题可用属性、装备成长分类等规则散落在多处，新增一个属性需要反复补名单。

## 设计

### 1. 服务端建立单一属性注册表

新增 `server/src/services/shared/characterAttrRegistry.ts`，集中定义：

- 属性 key
- 展示中文名
- 是否百分比属性
- 是否允许进入功法被动
- 功法被动模式：`flat / percent / multiply`
- 是否允许标题效果使用
- 装备成长分类：攻击 / 防御 / 生存

由该模块导出多组共享集合，供现有模块复用：

- 比率属性集合
- 功法被动语义字典
- 功法被动模式字典
- 标题效果允许 key
- 装备成长分类集合
- 属性中文名映射

### 2. 新增属性 `jianfantan`

语义：

- 中文：反弹伤害减免
- 类型：百分比属性
- 功法被动：允许
- 装备成长分类：防御

作用范围：

- 技能 Buff `reflect_damage`
- 套装/词条 `damage_type: reflect`

不影响：

- 普通物理/法术/真实伤害
- DOT/HOT
- 回响 `echo`

### 3. 战斗结算规则

反弹伤害统一先算原始反弹值，再应用减免：

`finalReflectDamage = floor(sourceDamage * rate * max(0, 1 - defender.jianfantan))`

这里的 `defender` 指“反弹伤害的承受者”，也就是原攻击方。

统一在 `reactiveDamage.ts` 收口，避免技能反弹与套装反弹各写一套减免逻辑。

### 4. 前端展示收口

复用并扩展 `client/src/pages/Game/shared/attrDisplay.ts`，把这些现有重复定义尽量改为读取同一入口：

- 属性中文名
- 百分比属性集合
- rating 基础属性集合
- 角色/装备展示排序

`Game/index.tsx`、`itemMetaFormat.ts`、`techniquePassiveDisplay.ts` 等已有重复映射改为复用该共享入口。

## 涉及范围

服务端：

- 属性注册表与共享约束
- 角色/伙伴/怪物/战斗快照属性结构
- 反弹伤害结算
- 信息面板展示

客户端：

- Socket/DTO 类型
- 人物面板
- 伙伴/装备/功法展示共享映射

## 不做的事

- 不引入旧字段兼容或别名字段
- 不改现有 `echo` 机制
- 不做无关属性系统重构
- 不执行任何 Git 操作
