# 九州修仙录 - 功法玩法策划

> 版本：v0.1  
> 日期：2026-01-30  
> 关键词：功法、层数、材料、灵石、经验、回合制技能、流派构筑

## 1. 玩法目标

- 用“功法层数”作为核心成长线，支撑长期挂机产出与回合制战斗策略深度。
- 通过“材料 + 灵石 + 人物经验”的三重消耗，建立可控的成长节奏与经济闭环。
- 让“修炼层数 → 解锁/强化技能”形成清晰可感知的成长反馈，并支持多流派构筑与克制关系。

## 2. 核心循环

1) 战斗/探索（挂机、秘境、副本、PVP）  
2) 获得：功法材料、灵石、人物经验（exp）  
3) 修炼功法：消耗材料 + 灵石 + exp 升层  
4) 解锁/强化技能与被动 → 构筑变化  
5) 进入更高难度内容 → 更高产出 → 循环

## 3. 功法系统概述

### 3.1 层数规则

- 功法层数：1～9 层，但“层数上限”由品质决定（最高品质才会有 9 层）。
- 品质与层数上限建议：
  - 黄：3 层
  - 玄：5 层
  - 地：7 层
  - 天：9 层
- 每一层提供：
  - 被动加成（属性/抗性/机制类）
  - 解锁 0～1 个主动技能，或强化 1 个已有技能
  - 关键层提供“机制跃迁”（例如：获得新资源、连携规则、领域效果等）
    - 天品：建议 3/6/9
    - 地品：建议 3/5/7
    - 玄品：建议 3/5
    - 黄品：建议 3

### 3.2 分类与定位（建议）

为保证构筑空间与回合制对战可读性，功法可按“定位”划分，并用标签系统扩展：

- 心法：资源循环（灵气/真元）、生存、抗性、回合节奏
- 武技：物理输出、爆发、追击、破甲、反击
- 法诀：法术输出、DOT、控制、领域、驱散
- 身法：速度、闪避、解控、反制、保命
- 辅修：治疗、护盾、增益、召唤、团队支援

标签建议：流派（剑/刀/拳/法阵/符咒）、元素（金木水火土/阴阳）、距离（近战/远程）、目标（单体/群体/随机/链式）、机制（蓄力/连携/召唤/领域/反制）。

说明：
- 武器不限定固定类型，功法与技能不依赖“必须装备某类武器”才能释放；武器更多作为数值与特效来源。

## 4. 装备与构筑（建议）

为避免“全学全带”导致的强度膨胀，建议采用“装备生效”规则：

- 装备槽：主功法 1 个 + 副功法 3 个（共 4 个生效）
- 生效规则：仅“装备中的功法”提供被动与技能；未装备功法不提供战斗效果
- 被动加成比例：
  - 主功法：被动属性/机制加成按 100% 生效
  - 副功法：被动属性/机制加成按 30% 生效
- 技能槽：总计最多 10 个主动技能槽（从装备功法已解锁技能中选择装配）
- 同名/同类增益叠加：默认不叠加（取最高或刷新持续时间），避免无限堆叠

备注：
- 主/副功法加成比例仅作用于“被动加成”。主动技能的释放与效果不因主/副区分而打折，避免出现“副功法技能不可用/不值得点”的问题。

## 5. 功法升级规则

### 5.1 升级消耗

每次升层固定消耗三类资源：

- 灵石：货币消耗
- 材料：功法材料（通用 + 定向）
- 人物经验（exp）：从角色经验池中扣除，用于“修炼投入”

说明：
- exp 作为可积累资源参与消耗，能够把挂机产出与成长节奏绑定；exp 不足则不可升级。
- 升级成功率建议为 100%（更清晰、更少挫败）。如需加入挑战，可放在关键层的“突破任务”，但不要引入随机失败吞资源。

### 5.2 升级前置

- 前置层：必须达到当前层（n）才可升到（n+1）
- 境界门槛：达到最低境界（与项目境界体系一致）
- 可选门槛：指定副本通关/任务完成/宗门权限（用于高级功法）

### 5.3 消耗曲线（建议）

建议采用“温和指数”曲线，保证前期爽感、后期拉开差距：

- 灵石：约按层数 1.6～1.9 倍递增
- 材料：数量递增 + 品质递增（普通 → 精良 → 稀有 → 史诗）
- exp：与灵石同阶递增，但略高于灵石以控制节奏

示例（仅作数值结构参考，可后续调参）：

说明：下表为“天品 9 层”示例；其他品质功法仅计算到自身层数上限。

| 升层 | 灵石 | exp | 通用残页 | 定向材料 |
|---|---:|---:|---:|---|
| 1→2 | 200 | 300 | 5 | 0 |
| 2→3 | 500 | 800 | 10 | 2 |
| 3→4 | 1,200 | 2,000 | 18 | 4 |
| 4→5 | 2,800 | 4,800 | 28 | 6 |
| 5→6 | 6,000 | 10,000 | 40 | 9 |
| 6→7 | 12,000 | 22,000 | 55 | 12 |
| 7→8 | 24,000 | 46,000 | 72 | 16 |
| 8→9 | 50,000 | 95,000 | 90 | 20 + 瓶颈材料 |

瓶颈材料建议：悟道玉简/道韵晶核（用于 8→9 或 6→7、8→9）。

### 5.4 材料体系（建议）

- 通用材料：
  - 功法残页：所有功法升层通用消耗
  - 灵墨/灵木/兽骨/妖血：按大类掉落，保证获取稳定
- 定向材料：
  - 元素材料：火灵砂/水灵髓/金灵铁/木灵髓/土灵髓
  - 流派材料：剑意石/拳罡骨/符箓纸/阵纹石
- 瓶颈材料：
  - 悟道玉简/道韵晶核：Boss、周常、宗门兑换、拍卖行

来源设计：
- 挂机小怪：通用材料为主（稳定产出）
- 精英/首领：定向材料（可控差异化）
- 秘境/副本：定向材料 + 瓶颈材料（目标导向）
- 宗门商店：补齐短板，防止卡死
- 市场交易：允许玩家间流通，形成经济玩法

## 6. 回合制技能设计框架

### 6.1 技能基础参数

每个主动技能建议包含：

- 目标类型：单体 / 群体 / 列 / 链 / 自身 / 友方
- 资源消耗：灵气（qi）/ 其他派生资源（如剑意、法印）
- 冷却：按回合计（0～6）
- 释放优先级：普通 / 追击 / 反击 / 插入（用于反制类技能）
- 命中与豁免：命中/闪避/控制抗性/驱散抗性等
- 效果列表：伤害、治疗、护盾、增益、减益、控制、召唤、领域

### 6.2 效果类型（覆盖主要玩法）

为了“考虑所有可能玩法”，建议至少覆盖以下效果族群，并给出清晰的克制链：

1) 输出体系  
- 物理/法术/真实伤害  
- 单体爆发/范围清场/持续伤害（DOT）  
- 斩杀与阈值伤害（例如目标低于 30% 生命伤害提升）  

2) 生存体系  
- 护盾（吸收伤害）、减伤、格挡、反伤  
- 治疗（直接/持续）、吸血、护体（免控/免伤一次）  

3) 控制与反控制  
- 硬控：眩晕（跳过行动）、冻结（无法行动且易伤）  
- 软控：沉默（禁法）、缴械（禁武）、迟滞（降低速度/命中/伤害）  
- 解控：净化（移除控制/减益）、免控（若干回合）  
- 递减机制：同类硬控在短时间内持续递减（避免无限控）  

4) 增益与减益  
- 破甲、易伤、减疗、减速、封灵（降低灵气恢复/上限）  
- 强化：攻击/防御/命中/闪避/暴击/冷却缩减等  
- 规则：同名不叠加（刷新），同类上限叠加层数（例如最多 5 层）  

5) 资源与节奏  
- 回气/偷气/燃气（抽干对方资源）  
- 蓄力：本回合准备，下回合释放更强效果（可被打断）  
- 追击/反击：在特定条件触发额外结算（例如闪避后反击）  

6) 构筑型机制  
- 标记/引爆：先叠“印记”，再用引爆技能获得高收益  
- 领域：持续 N 回合改变规则（例如火域：火系伤害+、治疗-）  
- 召唤：召唤物独立回合行动，继承属性比例，提供坦/奶/控/输出  

### 6.3 PVE/PVP 平衡约束（建议）

- Boss 机制：对硬控免疫或强衰减；引导玩家用破甲/DOT/反制等策略
- 反拖时长：PVP 可加入“回合数加深易伤/治疗衰减”避免无限苟活
- 驱散与免疫：驱散/免疫要有成本与冷却，避免一键否定流派

## 7. 解锁节奏建议（随品质层数上限）

建议“每层都有获得感”，并按“层数上限”设置关键跃迁点：

- 第 1 层：入门技能（基础输出/基础回气）+ 基础被动
- 第 2 层：补齐短板（生存/控制/驱散三选一）或技能小强化
- 中段关键层（取 maxLayer 的 60%～80%）：机制成型（标记/引爆、领域、资源闭环、召唤体系等）
- 最高层（maxLayer）：终极特性（决定流派上限的被动或终结技）

按品质上限拆分的示例：
- 黄（3层）：1层入门 → 2层补齐短板 → 3层小机制/小终式
- 玄（5层）：1层入门 → 2层补齐短板 → 3层小机制 → 4层强化 → 5层终式
- 地（7层）：在 3/5/7 层形成递进（机制雏形 → 机制成型 → 终式）
- 天（9层）：在 3/6/9 层形成递进（机制雏形 → 机制成型 → 终式）

## 10. 功法与战斗系统集成

### 10.1 技能来源与加载

战斗开始时，系统需从以下来源收集角色可用技能：

1. 装备功法解锁的技能（主功法 + 副功法）
2. 技能槽中已装配的技能（最多10个）
3. 装备附带的技能（如有）
4. 消耗品临时技能（如有）

加载流程：
```
角色ID → 查询 character_technique（已装备功法）
       → 查询 technique_layer（当前层数解锁的技能ID）
       → 查询 character_skill_slot（已装配技能）
       → 查询 skill_def（技能详细定义）
       → 构建 BattleSkill[]
```

### 10.2 被动属性计算

战斗属性快照时，需叠加功法被动：

```ts
// 被动加成计算
function applyTechniquePassives(
  baseAttrs: BattleAttrs,
  techniques: CharacterTechnique[]
): BattleAttrs {
  const result = { ...baseAttrs };
  
  for (const tech of techniques) {
    const layers = getTechniqueLayers(tech.techniqueId, tech.currentLayer);
    const ratio = tech.slotType === 'main' ? 10000 : 3000; // 主100%/副30%
    
    for (const layer of layers) {
      for (const passive of layer.passives) {
        // passive.value 已是万分比
        const actualValue = Math.floor(passive.value * ratio / 10000);
        result[passive.key] = (result[passive.key] || 0) + actualValue;
      }
    }
  }
  return result;
}
```

### 10.3 技能冷却与切换规则

冷却规则：
- 冷却以"回合"为单位，双方各行动一次 = 1回合
- 回合结束时（双方都行动完）统一扣减冷却
- 冷却缩减属性（lengque）影响实际冷却：`actualCD = ceil(baseCD * (10000 - lengque) / 10000)`

功法切换规则：
- 战斗中不可切换功法
- 非战斗状态下切换功法有冷却（建议30分钟）
- 切换后技能冷却重置

### 10.4 技能升级机制

技能随功法层数自动强化：

```ts

  skillId: string;

  changes: {
    coefficient?: number;  // 系数增加
    fixedDamage?: number;  // 固定值增加
    addEffect?: SkillEffect;  // 新增效果
    targetCount?: number;  // 目标数变化
  };
}
```

## 11. 组队与功法协同

### 11.1 组队战斗中的功法

组队战斗时，每个玩家独立计算自己的功法被动与技能：

- 主/副功法被动：仅作用于自身
- 技能效果：根据技能目标类型决定作用范围
- 团队增益技能：可作用于队友（如"终式·大吐纳"）

### 11.2 功法流派搭配建议

为组队提供构筑参考：

| 定位 | 推荐功法类型 | 核心职责 |
|---|---|---|
| 主T | 心法（生存）+ 身法（解控） | 吸引仇恨、承伤、控场 |
| 输出 | 武技/法诀（输出）+ 心法（续航） | 持续/爆发伤害 |
| 辅助 | 辅修（治疗）+ 心法（资源） | 治疗、护盾、增益 |
| 控制 | 法诀（控制）+ 身法（保命） | 控制、减益、打断 |

### 11.3 功法连携（可选扩展）

未来可扩展的组队机制：

- 同门功法：队伍中多人装备同一功法时，触发额外被动
- 流派共鸣：特定流派组合触发团队增益（如：剑阵、五行阵）
- 连携技：特定技能组合触发追加效果

## 12. 数据库表设计

### 12.1 功法定义表 (technique_def)

```sql
CREATE TABLE IF NOT EXISTS technique_def (
  id VARCHAR(64) PRIMARY KEY,                         -- 功法ID，如 'tech-tunajue'
  name VARCHAR(50) NOT NULL,                          -- 功法名称
  type VARCHAR(20) NOT NULL,                          -- 类型：心法/武技/法诀/身法/辅修
  quality VARCHAR(10) NOT NULL,                       -- 品质：黄/玄/地/天
  max_layer INTEGER NOT NULL,                         -- 最大层数：3/5/7/9
  required_realm VARCHAR(50) NOT NULL,                -- 最低境界要求
  tags TEXT[] DEFAULT '{}',                           -- 标签数组
  description TEXT,                                   -- 功法描述
  icon VARCHAR(255),                                  -- 图标路径
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

COMMENT ON TABLE technique_def IS '功法定义表';
```

### 12.2 功法层级表 (technique_layer)

```sql
CREATE TABLE IF NOT EXISTS technique_layer (
  id SERIAL PRIMARY KEY,
  technique_id VARCHAR(64) NOT NULL REFERENCES technique_def(id),
  layer INTEGER NOT NULL,                             -- 层数 1-9
  
  -- 升级消耗
  cost_spirit_stones INTEGER NOT NULL DEFAULT 0,      -- 灵石消耗
  cost_exp INTEGER NOT NULL DEFAULT 0,                -- 经验消耗
  cost_materials JSONB DEFAULT '[]',                  -- 材料消耗 [{itemId, qty}]
  
  -- 被动加成
  passives JSONB DEFAULT '[]',                        -- [{key, value}] value为万分比
  
  -- 技能解锁/强化
  unlock_skill_ids TEXT[] DEFAULT '{}',               -- 本层解锁的技能ID
  upgrade_skill_ids TEXT[] DEFAULT '{}',              -- 本层强化的技能ID
  
  -- 前置条件
  required_realm VARCHAR(50),                         -- 本层境界要求（可选）
  
  UNIQUE(technique_id, layer)
);

COMMENT ON TABLE technique_layer IS '功法层级表';
```

### 12.3 技能定义表 (skill_def)

```sql
CREATE TABLE IF NOT EXISTS skill_def (
  id VARCHAR(64) PRIMARY KEY,                         -- 技能ID
  name VARCHAR(50) NOT NULL,                          -- 技能名称
  description TEXT,                                   -- 技能描述
  icon VARCHAR(255),                                  -- 图标路径
  
  -- 来源
  source_type VARCHAR(20) NOT NULL,                   -- innate/technique/equipment/item
  source_id VARCHAR(64),                              -- 来源ID（功法ID/装备ID）
  
  -- 消耗与冷却
  cost_lingqi INTEGER DEFAULT 0,                      -- 灵气消耗
  cost_qixue INTEGER DEFAULT 0,                       -- 气血消耗
  cooldown INTEGER DEFAULT 0,                         -- 冷却回合数
  
  -- 目标
  target_type VARCHAR(20) NOT NULL,                   -- self/single_enemy/single_ally/all_enemy/all_ally/random_enemy/random_ally
  target_count INTEGER DEFAULT 1,                     -- 目标数量
  
  -- 伤害/治疗
  damage_type VARCHAR(20),                            -- physical/magic/true
  element VARCHAR(10) DEFAULT 'none',                 -- 元素：none/jin/mu/shui/huo/tu
  coefficient INTEGER DEFAULT 0,                      -- 系数（万分比）
  fixed_damage INTEGER DEFAULT 0,                     -- 固定伤害/治疗值
  
  -- 效果列表
  effects JSONB DEFAULT '[]',                         -- SkillEffect[]
  
  -- 触发类型
  trigger_type VARCHAR(20) DEFAULT 'active',          -- active/passive/counter/chase
  
  -- AI优先级
  ai_priority INTEGER DEFAULT 50,                     -- AI使用优先级 0-100
  
  -- 技能升级定义
  upgrades JSONB DEFAULT '[]',                        -- SkillUpgrade[]
  
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

COMMENT ON TABLE skill_def IS '技能定义表';
```

### 12.4 角色功法表 (character_technique)

```sql
CREATE TABLE IF NOT EXISTS character_technique (
  id SERIAL PRIMARY KEY,
  character_id INTEGER NOT NULL REFERENCES characters(id),
  technique_id VARCHAR(64) NOT NULL REFERENCES technique_def(id),
  
  current_layer INTEGER DEFAULT 1,                    -- 当前层数
  slot_type VARCHAR(10),                              -- 装备槽：main/sub/null(未装备)
  slot_index INTEGER,                                 -- 副功法槽位 1-3
  
  acquired_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,    -- 获得时间
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  
  UNIQUE(character_id, technique_id)
);

CREATE INDEX idx_char_tech_char ON character_technique(character_id);
CREATE INDEX idx_char_tech_slot ON character_technique(character_id, slot_type);

COMMENT ON TABLE character_technique IS '角色功法表';
```

```sql
CREATE TABLE IF NOT EXISTS character_skill_slot (
```
