# 九州修仙录 - 签到与活动系统设计

> 版本：v0.1  
> 日期：2026-02-01  
> 目标：设计签到、累计登录、限时活动等运营系统。

## 1. 设计原则

- **日常粘性**：每日签到培养登录习惯
- **长期目标**：累计奖励提供长期追求
- **运营灵活**：活动可热更新配置
- **公平合理**：免费玩家也能获得基础奖励

## 3. 数据结构

### 3.1 签到配置表 (sign_in_config)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | varchar(64) | 配置ID |
| type | varchar(16) | 类型（daily/cumulative） |
| day | int | 天数 |
| rewards | jsonb | 奖励配置 |
| vip_bonus | jsonb | VIP额外奖励 |
| enabled | boolean | 是否启用 |

### 3.2 角色签到记录表 (character_sign_in)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigserial | 主键 |
| character_id | int | 角色ID |
| month | varchar(7) | 月份（2026-02） |
| signed_days | int[] | 已签到日期 |
| cumulative_days | int | 累计天数 |
| makeup_used | int | 已用补签次数 |
| claimed_cumulative | int[] | 已领累计奖励 |

| enabled | boolean | 是否启用 |

### 4.3 活动进度表 (character_event_progress)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigserial | 主键 |
| character_id | int | 角色ID |

| progress | jsonb | 进度数据 |
| claimed_rewards | jsonb | 已领奖励 |

## 7. API设计

```
GET /api/sign-in/status      # 签到状态
POST /api/sign-in/sign       # 签到
POST /api/sign-in/makeup     # 补签
POST /api/sign-in/claim      # 领取累计奖励

GET /api/event/list          # 活动列表
GET /api/event/:id           # 活动详情
POST /api/event/claim        # 领取活动奖励

GET /api/online/rewards      # 在线奖励状态
POST /api/online/claim       # 领取在线奖励
```
